## 运动重定向介绍

运动重定向：一种形态（一般为人）的运动数据转换到另一种运动形态（一般为机器人）上的技术，目的是完成数据迁移，让目标的运动形态能用这些数据进行学习

常见挑战：运动学差异，结构上的不匹配难以通过简单的数据映射来修正；动力学约束，机器人映射后的数据要满足物理定律与约束

## 运动重定向方法
### 显式优化方法
#### PHC——Perpetual Humanoid Control（持续人形控制）
PHC结构：Human Motion Dataset -> Motion Encoder -> Motion Representation -> RL Controller -> Humanoid Simulator -> Physics Feedback

核心思想1：Motion Piror

给一个人类动作先验，让rl在这附近探索，提高效率（先训练一个人体运动分布模型）

核心思想2：奖励函数设计

pose reward & velocity reward & root reward(奖励身体整体移动) & keypoint reward & motion prior reward（一个判别器看像不像人类，符不符合motion prior）& style reward & balance reward & energy reward & termination reward

核心思想3：PMCP——Progressive Multiplicative Control Policy（渐进式乘法控制策略）

对于多个task（如walking,running）先训练walking,得到策略头 $\pi_1$ ，再冻结策略头 $\pi_1$ ，根据running训练 $\pi_2$ （注意running的策略头是$\pi_1$$\pi_2$相乘）， 最后的策略是根据不同task激活不同策略头，让这些策略头的概率分布连乘得到最终的操作的概率分布。

核心思想4：就是说SMPL用一套骨架去拟合人类动作， retargeting把人体骨架点做映射，然后我们的要求是对某些关键节点让它位移相同，轨迹相同，这里用到IK去反解机器人的关节角，并作为PHC后续RL训练的动作先验

一个比较有趣的idea：机器人的能力增长具有层次性，比如你要学会平衡，站起来，再学会走路，再学会跑，所以PMCP比较适合机器人

#### Mink等高性能IK求解器
利用稀疏雅可比与高效迭代求解显著加速批量重定向过程（我也不知道这是啥，反正就加快IK求解了）

#### HumanMimic
同PHC很像，基本思路还是运动重定向后，强化学习训动作

核心思想1：primitive skeleton

把人和机器人之间搭建一个primitive skeleton（对骨长等做归一化处理），以学到更通用的运动（不是绝对位置，而是相对位置和更整体的运动，比如机器人走路不一定像人一样走80cm，而是比如机器人腿长的百分之多少）

核心思想2：Wasserstein Adversarial Imitation Learning

在RL的reward函数中，加入一项GAN的损失函数，让判别器来判断机器人的动作是不是又人类运动的风格，因而能保持运动风格与速度跟踪的同时实现自然步态与无缝速度过渡。

用Wasserstein是因为这个比较稳定，不会出现梯度消失的问题

#### OKAMI框架
网络结构：人类视频 -> VLM提取任务信息 -> constraint generator生成各种约束 -> 数值优化器求解robot trajectory -> 作为训练数据进行behavior cloning

核心思想：抛弃之前训练神经网络解决运动重定向的思路，改为约束满足问题

一个注意点：求解出来robot trajectory为什么不直接用，而要让神经网络BC下？
这是因为前面求解轨迹太慢了，不满足robot运动的实时性

#### TWIST框架

核心思路：

训练阶段：利用Explicit Mapping 和 IK 在线将输入帧映射到机器人上，产生reference motion, 之后再训练阶段先做BC，再做RL。

实际部署：利用Explicit Mapping 和 IK 在线将输入帧映射到机器人上，产生reference motion, 之后再根据policy做一次前向推理，做出motor command,让机器人运作

解决问题：机器人如何实时模仿一个正在运动的人？

#### OmniRetarget
有趣的想法：之前的运动重定向就是把人的运动或结构变为机器人的结构，但日常生活中很多操作比如搬箱子，不仅要考虑人的运动，还要考虑人和箱子之间的相对关系。作者提出，Human Motion ≠ Pose。真正应该保持的是interaction,而不是pose

核心创新：Interaction Mesh（大量几何约束组成的网络）

优化目标变成优化interaction error,以保持相对几何关系，而不是之前absolute pose

生成多样轨迹：优化interaction error可能出现很多解

特别适合Loco-Manipulation（一边动，一边操作）：因为Loco-Manipulation需要满足很多约束，而Interaction Mesh天然就是约束

Zero-shot Sim-to-Real:因为interaction更稳定，reference泛化性更好，不用重新修改，实现zero-shot，而过去以pose为导向的retarget,可能会出现穿模等问题，可能还需要进一步训练

#### GMR
核心论证：重定向质量对下游策略学习的决定性影响（Garbage in -> Garbage out）
核心创新1：Non-uniform Local Scaling（对身体不同部分分别进行缩放，在一开始的几何映射阶段）

核心创新2：Two-stage IK

先求解rotation，确立机器人的姿态，再对机器人求解root translation

启发：对耦合的求解的问题要拆分来求

### 隐式优化方法
#### 深度卷积神经网络的通用框架（将运动重定向视为低维流形空间内的约束优化问题）
核心思想：由于人类运动的秩序性，我们可以把其原来的高维运动空间，转化为低维的latent space（隐空间），在里面求解一些问题。这样的好处是我在隐空间中求解出来的向量，用decoder生成的动作天然满足人类运动风格，而IK反解出来的解由于自由度的冗余，可能不符合人类形态要求

具体操作：对关节角和时间组成的矩阵，用简单的CNN编码器（encoder)求解隐空间，在用decoder还原出来，要求最小化生成和原来的差异

怎么判断latent space满不满足要求？一是要求latent space满足低维特征，二是要求最后的loss比较小

怎么在隐空间中求解约束优化？

首先我们设约束函数C(x),约束条件就是C(x) = 0（比如手部约束就可以是 C(x) = abs（pos_x_hand - pos_target））,然后要求解的就是latent space中最小化 C(Decoder(z)) 的 z 这就用梯度下降来求解，而这样求出来的z的decoder(z)就是我们需要的，满足人类运动风格的解

注意点：在实际操作时常常会 对原本的约束函数C 加上一项 $\lambda$ $|z-z_0|^2$ 以保证求解出来的动作距离原动作相差不远

局限：latent隐空间中知道运动学，没有动力学

#### Pose-to-Motion 框架
适用场景：source domain有大量的运动数据，root domain只有少量static pose，如何生成root domain的motion数据（对应现实中人类动作视频数据较多，而机器人的motion数据较少）

网路结构：motion sequences(source domain) -> Generator G -> a sequence of poses and root transformations in target domain -> Generator F -> a motion sequence in the source domain

GAN损失函数方面：一个判别器判别pose像不像，一个判别器判别学回去的source motion想不想，还有一个cycle_loss 要求F(G(x)) 约等于 x

有趣的idea: pose 判别器要求生成的 pose满足生成的target motion(其实是target pose + target transformation)中的pose和真实分布，这个cycle_loss让我们不会去学一个奇怪的映射，保证动作间的时序性

接下来的问题：如何求解root transformation?

论文指出：如果root transformation没有约束，会产生：foot sliding，jitter
引入约束：Contact consistency loss（脚接触地面时，速度应该为0）+ End-effector consistency loss(速度应该与root 到末端的骨骼链长度成比例） + Relative End-effector Loss（相对位置变化也应该成比例）

这里速度和位移都是包含root transformation的全速度和全位移（重点是推回root轨迹，所以可能略有重复），其中v是通过相邻pose差分得到的。速度约束是局部的，位移约束是全局的。

#### 基于统一动态图网络的跨结构运动重定向框架（图卷积没学过，口胡一下算了）

核心idea:运动语义和拓扑语义是可以区分的，运动不是依附于骨骼的。比如人和狗骨骼结构不用，却都有走路的动作语义

具体做法：

动态图学习动作中的动态关节关系，在后面的motion encoder和topology encoder

source motion -> motion encoder -> motion latent 

target skeleton -> topology encoder -> topology latent

motion latent + topology latent -> decoder -> target motion

注意：motion encoder, topology encoder, decoder, 动态图变换都有良好的泛化性，可以实现zero-shot

#### IKMR
Human Motion -> Graph Encoder -> human latent space -> Mapping Network -> robot latent space -> Graph decoder -> robot motion -> RL refinement -> physically feasible motion

其中mapping network需要大量human motion 与 robot motion 的配对数据训练

怎么判断encoder好不好？
1. 语义没有缺失，decoder还可以复原
2. 跨域一致性，human latent和robot latent应该关注共有的东西，而不是human latent关注骨骼长度，robot latent关注电机编号之类的
3. latent interpolation 跑步和走路的插值应该出现慢跑和快走
4. 下游任务性能

## 步行、跑步等基础运动技能学习
### 传统方法
#### ZMP：Zero Moment Point，零力矩点
核心想法：只要机器人脚底与地面的合力作用点（ZMP）落在支撑区域内，机器人就不会翻倒。

流程：目标步态->规划脚的位置、轨迹->计算身体质心轨迹(CoM)->保证ZMP始终在脚底范围内->逆运动学求关节角->电机控制

#### MPC（Model Predictive Control）
思想：每隔一小段时间，根据当前状态预测未来，然后优化未来一段时间的控制。预测十步，执行一步，然后再预测

问题：依赖精确建模，优化问题计算成本大

### 现代方法
#### rl想法
输入加一个command防止训练训练训到后面忘记了前面技能
#### DWL（Denoising World Model Learning，去噪世界模型学习
想法：传统RL只知道当前状态，不知道未来状态

改进: DWL用denoising训练world model得到未来状态的概率分布，作为额外变量输入给RL policy，或者做MPC

#### Humanoid-Gym 框架
本质上就是做大量的随机操vx作，实现zero-shot sim to real

框架：Isaac Gym -> 大规模并行训练(PPO Reinforcement Learning)->Domain Randomization->Observation Noise->Random Push->Learned Policy->真机 XBot

#### PIM（Perceptive Internal Model）
idea:把未来地形编码成一个隐变量，再交给RL Policy。（人都是根据过去和当下做出的reactive motion，人在上楼梯的时候还会提前看，做出决策）

具体操作depth camera采集高度信息，得到elevation map（高度矩阵），再用PIM 提取出来terrain latent这个的向量，输入到RLpolicy中

#### 一种混合控制架构
原因：Model-based更精确，但不够鲁棒，神经网络不够精确，但很鲁棒，并且纯rl，在reward sparse的情况下很难训

思想：用 Model-based Planner 决定"应该怎么走"，给出reference motion, 再用 RL 学出的神经网络负责"稳稳地把它走出来"。

#### BeamDojo算法

核心解决问题：在sparse reward的环境下（比如在稀疏落脚点的环境下走），如何有效地训练robot?

结构：还是双层结构，一个global planner输出vx,vy,ω角速度等高层运动指令-> BeamDojo policy 输出足部落点和全身动作

Beamdojo policy的输入:机器人本体状态 + elevation map + velocity command

一些有趣的idea:
1. 怎么让机器人在稀疏落脚点上成功走路？加入一个reward——Sampling-based Foothold Reward（脚上装传感器，脚上接触点越多，奖励越大，自然会学得稳）
2. Double Critic:在这个环境下，关于速度，位移等的奖励是dense reward,关于落脚点的奖励是sparse reward。问题是两者同时训练，sparse reward会被dense reward给淹没,文章提出可以把其看为两个critic,分别进行训练（实际上一个训练走得稳，一个训练踩得准）
3. Two-stage RL. 问题：在稀疏落脚点的时候，很容易出现踩到石头没站稳，直接摔倒，过早结束训练的情况。针对这一问题，采用双阶段强化学习训练。第一阶段，在平地上接收任务地形的感知信息，从而避免误踩导致训练过早终止，同时仍然学习如何根据目标地形进行决策。第二阶段再上真环境，进行fine-tuning
4. 机器人中心LiDAR Elevation Map:就机器人中心实时建个图

####  Duke Humanoid鼓励被动动力学
先插一嘴，机器人减少能量主要有两种途径，一种是鼓励被动动力学，另一种是改变RL中奖励函数，惩罚高能量的动作

为利用被动动力学，设计了一个新的机器人结构：机器人 Duke。让Duke的脚靠的更近，这样质心更容易左右摆动，可以保持Straight Knee（直腿）（防止弯腿一直导致电机消耗能量）

训练两个policy:tracking + passive policy(奖励机器人不要频繁改变自身的自然运动）

## 平衡学习及跌倒恢复
### ASAP 框架
不识别真实世界的参数，而是学一个residual action model,一个可能只有两三层的模型修正

idea:和显微镜的粗细准焦螺旋很相似，其实就是先修正大的，再修正小的，这样一个想法
### Any2Track双阶段强化学习框架
idea:还是解耦的一个想法，就像之前认为运动可以拆解为pose + root transformation,这个认为应该用anytracker学习运动技能，再用一个anyadapter学习适应能力

anytracker:motion tracking RL,输入当前状态和reference motion,你要学会去模仿它，

anyadapter结合机器人状态和机器人状态的历史观测，输出enviroment embedding,这个作为额外输入输入到RL policy中，训练时端到端训练。

### CAP对抗学习机制
提高鲁棒性，就是做各种随机化，比如给一个推力什么的，然后这里就是用一个对抗学习，培养一个更会捣乱的老师。
